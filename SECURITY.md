# Security Policy

## Reporting a Vulnerability

We take the security of AssetFlow seriously. If you discover any security vulnerability in this project, please do NOT report it via public issues or discussions.

Instead, please send an email to: **security@example.com** (replace with security email for production).

Please include the following details in your report:
* Type of vulnerability (e.g., SQL injection, XSS, RBAC bypass, CSRF)
* Detailed steps to reproduce the vulnerability (including payloads or API endpoints)
* Potential impact of the vulnerability
* Any proposed fixes or mitigation strategies

We will review your submission and respond within 48 hours to coordinate a disclosure and patch schedule.

## Supported Versions

Only the latest release version on the `main` branch is actively supported with security updates. We recommend always running the latest version.

| Version | Supported          |
| ------- | ------------------ |
| v1.0.x  | :white_check_mark: |
| < v1.0  | :x:                |
